## E. The REWRITE — entitlement-based access (the load-bearing change)

haworks `GetMediaUrl.cs:42` authorises by **ownership**: `currentUser.UserId == file.OwnerId`. **This is wrong for a store** — the buyer is not the owner. Replace the predicate with the `IDownloadAuthorizer` seam:

```csharp
// BEFORE (haworks): if (!string.Equals(file.OwnerId, ownerId, StringComparison.Ordinal)) return Forbidden;
// AFTER:
if (!await _authorizer.CanDownloadAsync(requesterId, file.Id, ct))
    return Result.Failure<...>(new Error("Delivery.Forbidden", "No entitlement for this file."));
```

- Keep the `file.Status != Active → NotReady` check and the variant validator.
- **Do not implement `CanDownloadAsync` here.** Its body (check a live, non-revoked entitlement) belongs to the fenced `<House>.Entitlements` module. This module ships a `DenyAllDownloadAuthorizer` default so it builds and tests standalone; the consumer wires the real one.

## F. Bugs to fix + adaptations

1. **`ExpiresAt` divergence (bug).** `GetMediaUrl.cs:54` hardcodes `DateTime.UtcNow.AddMinutes(60)` while the URL uses `MediaStorageOptions.PresignedUrlExpiryMinutes`. Make both derive from the **same** value via `TimeProvider`, so the reported `ExpiresAt` never lies.
2. **Separate, short download TTL (adaptation).** haworks reuses one `PresignedUrlExpiryMinutes` (default **60**) for both PUT-upload and GET-download. Split into `UploadUrlTtl` (e.g. 60 min) and `DownloadUrlTtl` (default **3 min**). A paid download link must be short-lived.
3. **Dedupe (smell).** `MediaFile.MarkDeleted` just calls `MarkAsDeleted` — keep one.

## H. Config & secrets setup (executor: follow this exactly)

**Non-secret config** → `appsettings.json` `SecureDelivery` section (committed, no secrets):
```json
"SecureDelivery": {
  "Enabled": true,
  "BucketName": "packs",
  "Region": "auto",
  "ServiceUrl": "",            // empty = real AWS; set for Tigris/R2/LocalStack
  "DownloadUrlTtlMinutes": 3,
  "UploadUrlTtlMinutes": 60
},
"ClamAv": { "Host": "clamav", "Port": 3310 }
```

**Secrets** (`AccessKey`/`SecretKey`) — **never in appsettings.json, never in code**. Two supported paths, in order of preference:
1. **Production (preferred): no static keys.** Leave `AccessKey` empty → the AWS default credential chain (IAM role / instance profile / IRSA) is used automatically (lifted from `Program.cs:78-83`). Host grants the role S3 access.
2. **Dev / S3-compatible (Tigris/R2/LocalStack):** inject via **environment variables** (`SecureDelivery__AccessKey`, `SecureDelivery__SecretKey`) or **.NET user-secrets** (`dotnet user-secrets set "SecureDelivery:AccessKey" ...`). Never commit them.

**Fail-fast validation** (lift the intro-exchange pattern): `SecureDeliveryOptionsValidator : IValidateOptions<SecureDeliveryOptions>` throws at boot if `Enabled && BucketName` is empty, or if `ServiceUrl` is set but a key is missing in a non-IAM setup. Wire with `.ValidateOnStart()`.

**Build-time guard:** the `<House>.Engineering` conformance check must fail the build if an S3 key literal (`AKIA…`, a secret-looking string) appears in any `appsettings*.json` or `.cs`. (intro-exchange enforces the equivalent for the web; we extend it to .NET.)
